# Codex 任务卡模板

复制下面模板给 Codex 使用。每个任务尽量保持单一目标，避免一个 PR 同时做多件事。

```markdown
目标：

背景：

限制：
1.
2.
3.

需求：
1.
2.
3.

不做范围：
1.
2.
3.

验收标准：
1.
2.
3.

测试步骤：
1.
2.
3.
```

## 填写建议

- `目标`：一句话说明要完成什么。
- `背景`：说明为什么要做，帮助 Codex 判断取舍。
- `限制`：明确不要改哪些系统，例如随机算法、Auth、GCS、世界频道或 UI。
- `需求`：列出必须实现的行为。
- `不做范围`：列出本次不开放、不重构、不接入的部分。
- `验收标准`：写成可以逐项检查的结果。
- `测试步骤`：写出自动检查和手动验证方式。
