# random-choice-app Roadmap

这份路线图用于帮助后续 Codex 任务对齐方向，避免功能混乱或 PR 范围过大。

## 阶段 1：核心随机功能

- 食物、饮料、旅行、购物、娱乐号码和自定义随机池。
- 支持候选锁定、消费等级、预算提示和结果收藏。
- 保持随机功能轻量、日常、非赌博导向。

## 阶段 2：云端同步 / 世界频道

- 账号注册登录。
- 最近决定、收藏、上传记录云端同步。
- 世界频道基础聊天和图片消息。
- 移动端世界频道可用性优化。

## 阶段 3：反馈和 Bug 收集

- 更多菜单中的反馈入口。
- 用户反馈写入 `randomChoiceFeedback`。
- 前端 JS 错误、Promise 错误和关键 API 失败写入 `randomChoiceClientErrors`。
- 建立反馈分流、优先级和每周处理流程。

## 阶段 4：安全治理 / 举报 / 封锁

- API CORS、登录限流、上传权限和用户数据访问保护。
- 世界频道发言频率限制。
- 世界频道消息举报。
- 用户封锁和被封锁消息过滤。

## 阶段 5：好友系统隐藏铺垫

- 隐藏 feature flag，开发期不开放 UI。
- 账号 privacy 和 worldPreferences 默认结构。
- 好友申请、好友列表和 friendship 数据结构。
- 为后续私聊、好友圈和世界频道筛选预留安全边界。

## 阶段 6：PWA / App 封装

- 完善 manifest、icons、theme color 和移动端启动体验。
- 优化离线和弱网提示。
- 为后续封装成 App 保持内建 i18n 架构。

## 阶段 7：塔罗 App 实验

- 作为独立实验方向评估，不混入 random-choice-app 主线。
- 若启动，应新建独立仓库或独立产品分支。
- 不应影响选择困难 App 的核心体验和发布节奏。
